# Digital Transformation Prompt Template (Updated)

You can reuse the prompt structure below for future client pitches. I have updated it to explicitly request a multi-file GitHub Pages-ready architecture instead of a single file.

**THE PROMPT**
You are a senior full-stack developer and business consultant. I need you to produce a complete digital transformation package for the following company. Deliver each section as a separate, complete deliverable.

## TARGET COMPANY
- Company Name: [COMPANY NAME]
- Website: [CURRENT WEBSITE URL]
- Industry: [INDUSTRY]
- Location: [CITY, COUNTRY]
- Contact: [PHONE / EMAIL]

## MY COMPANY (the pitching firm)
- Name: [YOUR COMPANY NAME]
- Address: [YOUR ADDRESS]
- Contact: [YOUR PHONE / EMAIL]
- Registration: [BUSINESS REG / CREDENTIALS]

## STEP 1: INVESTIGATION
Fetch and analyse the target company's current website. Research their products, history, competitors, and digital gaps.

## STEP 2: FEASIBILITY STUDY & PITCH DOCUMENT
Produce a professional document (DOCX or PDF format) containing:
1. Executive Summary (problem, solution, payback)
2. Company Assessment
3. Proposed Solution (Website, ERP, Payments)
4. Use Cases
5. Technical Architecture
6. Investment & ROI
7. Implementation Timeline

## STEP 3: EXECUTIVE FRONTEND DEMO (GITHUB PAGES PACKAGE)
Build a complete, fully functional multi-file frontend application ready to be deployed to GitHub Pages. Do not build a single-file prototype. The architecture must include:
- `index.html`: Modern landing page with hero, about, and navigation.
- `products.html`: Commercial product catalogue with layout grids.
- `erp.html`: Executive dashboard showing KPI cards, production tracking tables, quotation generators, and Chart.js analytics.
- `css/style.css`: Dedicated styling utilizing a modern, Fluent Design aesthetic (acrylic effects).
- `js/app.js`: Centralized JavaScript handling dummy data objects (simulating the future database), LocalStorage session logic, and dynamic DOM rendering.

### Requirements for the Demo:
- Responsive, mobile-first design matching the company's brand colours.
- Fully functional UI (buttons trigger alerts simulating payment gateway popups, PDF downloads, etc.).
- The frontend must be ready for immediate backend integration (database schemas and APIs will be added if the proposal is accepted).

## DELIVERY FORMAT
- Provide the Feasibility Study document.
- Generate all frontend code files (`index.html`, `products.html`, `erp.html`, `css/style.css`, `js/app.js`) and bundle them into a structured ZIP file that mimics a standard web project repository.
- Provide 15-20 specific image search terms for royalty-free images.
